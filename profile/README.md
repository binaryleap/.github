Products
TenantsDB
Multi-tenant database orchestration platform.
Provisions isolated PostgreSQL, MySQL, MongoDB, and Redis databases for every customer. No tenant_id columns. No shared tables. Built for B2B SaaS at scale.

Meyou
AI companion with permanent memory.
A graph-based memory architecture for personal AI. Each user has their own database. Conversations, relationships, and context persist for years. Built on a structured memory model rather than RAG.
Website
GhostStash
Zero-knowledge encrypted storage.
Vault-based file storage where the operator cannot read your data. Phantom rotation across infrastructure nodes. End-to-end encryption with provable isolation.

OmniQL
Open-source universal query language.
A unified query AST that compiles to PostgreSQL, MySQL, MongoDB, and Redis. Maintained as a standalone Go library with parser, builder, and reverse translation.

How we build
We prefer owning infrastructure over renting it. Our stack runs on dedicated servers in Europe and Turkey, not hyperscalers. We write Go for backend systems, Flutter for cross-platform interfaces, and treat databases as a primitive worth understanding deeply.
We open-source what makes sense as standalone libraries, like OmniQL. We keep proprietary what represents a productized integration of many parts, like TenantsDB.
